# OpenClaw Skills Collection

这是一个全面的AI代理技能库，包含了一系列自动化工具和智能分析系统，专为OpenClaw AI代理平台设计。该项目整合了信息聚合、系统监控、金融分析、生活建议等多个领域的实用技能。

## 项目背景

随着AI代理技术的发展，我们致力于构建一套完整的自动化工具链，使AI代理能够更好地理解和应对复杂的现实世界任务。这些技能涵盖了从系统运维到市场分析，从新闻聚合到生活建议的各个领域，形成了一个综合性的智能工具集。

## 技能目录

### 📊 信息聚合与简报类

#### openclaw-product-tracker
**龙虾简报** - OpenClaw产品动态追踪技能，整合了GitHub、X平台和Moltbook论坛三个信息源，生成每日产品动态简报。通过多源数据融合，提供全面的产品发展洞察。

#### moltbook-daily-report
**Moltbook每日报告** - 自动化AI代理社区分析报告生成系统。从Moltbook API获取实时数据，分析热门话题，生成包含视觉热图和深度洞察的综合报告，并发布到Telegra.ph。

#### global-intel-brief
**全球情报简报** - 国际新闻源信息聚合与分析工具。自动抓取BBC、The Guardian、Al Jazeera、南华早报等主流国际新闻源，进行智能聚类和去重，生成结构化的全球情报简报。

#### tech-brief
**科技简报** - 科技新闻聚合工具。专注于AI大模型、芯片/硬件、造车新势力、互联网巨头等领域的信息收集与分析，自动生成结构化科技简报。

#### daily-tech-brief
**每日科技巨头战略简报** - 经过验证的每日运行科技简报工具。从多个权威科技新闻源获取信息，每天早上6:15自动运行，生成专业的科技简报并通过Telegraph发布。

#### news-aggregator-completed
**综合新闻聚合器** - 从8个主要信息源（Hacker News、GitHub Trending、Product Hunt、36氪、腾讯新闻、华尔街见闻、V2EX、微博）获取、过滤和深度分析实时内容。适用于日常扫描、科技新闻简报、金融更新和热门话题的深入解读。

### 🌤️ 天气与生活类

#### running-weather-advisor
**晨跑天气顾问** - 结合生理节律和天气数据的个性化跑步建议工具。通过计算身体、情感、智力周期，结合天气信息，提供个性化的晨跑指导。

#### weather-forecast-new
**简化天气预报** - 指定地点基本天气信息查询工具。提供快速响应的天气查询服务，避免长时间等待详细预报的问题。

#### weather-simple
**简单天气预报** - 快速响应的天气查询工具，提供指定地点的基本天气信息。

### 📈 金融与商业类

#### stock-analysis
**上市公司结构化分析** - 标准化的深度财务分析工作流。使用CIP分析法（Context-Impact-Price），将原始财务数据转化为具备逻辑性、显式化和可视性的专业投资报告。

### 🐦 社交媒体类

#### x-platform-brief
**X平台简报** - 个人X平台时间线内容获取与展示工具。获取并展示最新的个人X平台内容，跟踪趋势话题。

### 🔧 系统运维类

#### system-patrol
**系统巡检** - 全面系统健康度检查技能。检查硬件资源、网络安全、系统服务、安全状态等多个维度，生成系统健康度报告。支持Azure云环境监控，包括Azure实例元数据检查和详细的错误日志分析。

### 📋 日常仪式类

#### daily-review-ritual
**每日复盘仪式** - 日常回顾与总结工具，帮助AI代理进行每日反思和总结。

## 使用方法

每个技能都有独立的目录和配置文件。要使用某个技能，只需导航到相应目录并执行主脚本或按照该技能的README文件指示操作。

## 本地运行说明

所有技能都可以在本地正常运行，安全审查仅替换了代码中的示例API密钥为占位符，不影响本地功能。要使技能正常工作，需要配置真实的API密钥：

### API密钥配置

大多数技能需要配置API密钥才能正常工作，具体要求因技能而异：

- **DeepSeek API**: 用于AI分析功能
- **X平台认证**: 用于获取Twitter/X数据
- **Moltbook API**: 用于访问Moltbook社区数据
- **Telegra.ph API**: 用于发布文章到Telegra.ph
- **GitHub CLI**: 用于访问GitHub数据

API密钥应通过环境变量或安全配置文件设置，不要硬编码在脚本中。

## 安全最佳实践

- 将敏感信息存储在环境变量中，而不是脚本内
- 不要在版本控制系统中提交包含真实API密钥的文件
- 定期轮换API密钥
- 使用最小权限原则配置API访问
